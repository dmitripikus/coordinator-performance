# Permanent log-level changes for `llm-d-inference-gateway-istio`

Log-level changes that survive pod restarts, rollouts, and image updates.
Unlike the runtime path in [gateway-log-runtime.md](gateway-log-runtime.md),
these commit the level into Istio's control-plane state, so istiod bakes it
into the Deployment args on every reconcile.

The gateway pod is generated by the Istio Gateway API controller from a
`Gateway` resource — editing the Deployment directly gets overwritten on the
next reconcile. Persist via one of the options below instead.

## Option 1 — Annotate the `Gateway` (recommended)

Scopes the change to this one gateway. Applied by istiod on the next reconcile
of the Gateway; the gateway pod restarts once, then the level persists across
future restarts.

### Find the Gateway

```bash
kubectl get gateway -n dpikus-pd
```

### Patch it

```bash
kubectl annotate gateway llm-d-inference-gateway -n dpikus-pd \
  proxy.istio.io/config='
proxyLogLevel: info
componentLogLevel: "http:info,ext_proc:debug"
' --overwrite
```

Or edit the `Gateway` directly:

```yaml
apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: llm-d-inference-gateway
  namespace: dpikus-pd
  annotations:
    proxy.istio.io/config: |
      proxyLogLevel: info
      componentLogLevel: "http:info,ext_proc:debug"
spec: {...}
```

Fields:

- `proxyLogLevel` — overall Envoy level (`trace|debug|info|warning|error|critical|off`).
  Equivalent to the `--proxyLogLevel` arg.
- `componentLogLevel` — per-component overrides, comma-separated
  `component:level` pairs. Equivalent to `--proxyComponentLogLevel`.

Handy components: `http`, `router`, `ext_proc`, `connection`, `upstream`, `filter`.

### Verify

```bash
POD=$(kubectl get pod -n dpikus-pd \
  -l gateway.networking.k8s.io/gateway-name=llm-d-inference-gateway \
  -o jsonpath='{.items[0].metadata.name}')

kubectl get pod -n dpikus-pd $POD -o yaml | grep -A1 proxyLogLevel
kubectl exec -n dpikus-pd $POD -c istio-proxy -- \
  pilot-agent request POST 'logging'
```

### Revert

Remove the annotation:

```bash
kubectl annotate gateway llm-d-inference-gateway -n dpikus-pd \
  proxy.istio.io/config- --overwrite
```

## Option 2 — Mesh-wide default via `MeshConfig`

Sets the default for **every** Envoy in the mesh (all gateways, all sidecars).
Usually too broad; prefer Option 1 unless you actually want everything louder.

```yaml
apiVersion: install.istio.io/v1alpha1
kind: IstioOperator
metadata:
  name: control-plane
  namespace: istio-system
spec:
  meshConfig:
    defaultConfig:
      proxyLogLevel: info
      componentLogLevel: "http:info"
```

Apply with whatever tool your install uses (`istioctl install`, Helm,
GitOps). A pod-by-pod restart is required for existing workloads to pick up
new default args.

## Option 3 — Edit the Deployment directly (not recommended)

You *can* edit the generated `llm-d-inference-gateway-istio` Deployment's args:

```yaml
- --proxyLogLevel
- info
- --proxyComponentLogLevel
- misc:warn,http:info,ext_proc:debug
```

But the Gateway API controller owns this Deployment and will revert the change
on the next reconcile (Gateway spec update, istiod restart, controller
resync). Only useful for a very short-lived experiment where you don't want to
touch the `Gateway` resource — and even then, Option 1 is nearly always simpler.

## Pod restart behavior

Changing `proxy.istio.io/config` on a `Gateway` typically triggers a rolling
restart of the gateway Deployment (istiod updates the Deployment spec, which
rolls the ReplicaSet). Expect one pod restart per annotation change; steady
state is stable across future restarts.

## Which option to use

| Need                                                        | Option |
|-------------------------------------------------------------|--------|
| Debug this specific gateway on multiple bench runs          | 1      |
| Debug for a single run, then revert                         | runtime (see [gateway-log-runtime.md](gateway-log-runtime.md)) |
| Change default log level for the whole mesh                 | 2      |
| Emergency tweak, will clean up in minutes                   | runtime |
